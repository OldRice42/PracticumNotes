Introduction to Positioning Elements

Awesome! You’ve learned how to build grid layouts, mastered the BEM methodology, semantics and hover effects. You can now create websites that are easy to both use and maintain. Yet, sometimes it comes down to dealing with more complex design that has elements positioned differently. For example, elements may overlap one another or have fixed positioning relative to the page.

It takes special tricks to make something like that using flexbox, and this is when the position property comes to the rescue.

All elements on the page are interconnected and react to changes happening to the surrounding elements by default. Such behavior may be hindering. Controlling positioning of interconnected elements in the flow can help with that. There're several types of positioning. Later we’ll explain how all of these types work.

You’ll learn how to position elements on a page anywhere you want, create fixed menus, center popup windows on the screen and use hover effects that work even with videos.

Flow and Static Positioning

Browsers organize elements given to them in HTML code according to an internal set of rules. Every block is placed on a new line, while inline elements are grouped together. Objects don't overlap one another, meaning that if you apply a margin to one, the surrounding elements will be pushed away accordingly. This means that all the elements on our page are interconnected. It's like a room full of people who can all see each other, and react to each other's actions.

This connection between elements and their dependency on one another is referred to as flow. By default, all elements on a page are part of the flow.

The way in which elements are arranged in a layout is referred to as positioning. The CSS position property tells the browser where to place elements in the flow.

Static positioning is applied to pages by default, as described by the following property:

position: static;

This means that each element:

  1. is part of the flow, so all other elements know where it is and react when it's repositioned;
  2. only repositions itself visually when the margins are changed. From the browser's point of view, we don't actually move anything. Every block maintains its position in the flow, but we can change its size by modifying the margins.

Realative Positioning

When an element's position in the flow is adjusted relative to its normal position, it's said to have relative positioning. It has the position: relative declaration.

Here are some key things to remember about this type of positioning:

  1. Because an element stays in the flow, other elements still recognize its borders;
  2. The element doesn't move relative to its neighbors the way it would with static positioning, moving instead in relation to its own initial position as follows:
  • top: from the top border of the block;
  • right: from the right-hand border;
  • bottom: from the lower border;
  • left: from the left-hand border.

The last kind of positioning we're going to learn about is called absolute positioning. It’s one of the most common out there, and it's applied using the position: absolute declaration.

The first thing you need to know is that elements with absolute positioning are also pulled out of the flow, with the space they leave behind filled by the adjacent elements.

In contrast to fixed positioning, however, elements can behave differently depending on the situation.

The first case is when the parent element doesn't have any specific positioning (i.e. it has the default property position: static). In such cases, our block will go to the next parent in the hierarchy and try to position itself relative to its borders.

If all the parent elements have default positioning, it will position itself relative to the borders of the whole body element.
