Flexbox

Intro to Flexbox

Layout tools were originally developed for books and magazines, where the pages have fixed dimensions and structure. The primary purpose of layout tools is to ensure that all the elements on the page fit nicely together. In the case of print media, this mostly involves making sure that the text wraps around the images on the page.

This logic was carried over to webpage design, giving birth to the idea of floats. However, when developers started using floats to create websites, they ran into some problems. Changing a webpage's dimensions resulted in blocks getting moved around and text being cut off in places.

In trying to come up with a workaround for this issue, they started changing the behavior of inline elements, assigning them the display: inline-block; property in order to make them look like blocks. However, since they only looked like blocks, this created the new problem of gaps between them. This, in turn, required another workaround.

Eventually, there were so many workarounds, and layout tools had become so cumbersome that it became necessary to develop a completely new tool that would address all these complications. This is how the flexible box module, better known as flexbox, came about. It was created to make the process of arranging and aligning elements quick and easy.

Conversion to Flex Containers

Up until now, when we've written HTML code, the elements on the webpage have been displayed in the same order in which they appear in the code. This is no good if you want to make a news site or a shopping site. The most recent news items and the newest products should appear at the top of the user's feed. You also need the flexibility to change rows into columns for narrower mobile screens, as well as to align your content along the edge of the screen or in the center. Simply put, you need as much flexibility in your display as possible.

This one line of code provides the foundation for such freedom:

display: flex;

The display: flex property turns elements into flex containers and their child elements into flex items. The relationship between a container and its items is much like that of a soccer coach and their team. The coach sets the team up in a certain formation and tells them how to react in different situations. The players (flex items), in turn, can show their individual brilliance by virtue of their personal characteristics (properties) to decide exactly how they carry out the coach's orders. We'll learn how this all works in practice now.

Let's start off simple. We have a container with one child item and margin: auto as its only property.

The flex container doesn't govern the item itself, only the spacing around it. If we had a more specific property, such as margin-left: auto, the spacing would appear on the left and the item would sit on the right-hand edge of the container.

But margin: auto does something truly magical. It distributes the spacing equally around the item in all directions, putting it right in the center of its container, both in terms of width and height. This means we don't need the vertical-align property. And that's just the beginning!

Flex Containers with Multiple Items

When there are multiple items in a flex container, they all appear on the same line by default, regardless of type. You don't need to change the value of the display property to inline-block to achieve this. Since this is just one of the many possibilities provided by flexbox, they're much more widely used for organizing layouts than inline-block .

Flexbox Terminology

When it comes to flexbox, there are two main terms you need to know:

ul#Flex containers (parent elements): These are flexible spaces in which items are organized according to the rules of the container. ul#Flex items (child elements): These are the elements found inside the flex container. They conform to the rules of the container, as well as the rules applied to them specifically.
