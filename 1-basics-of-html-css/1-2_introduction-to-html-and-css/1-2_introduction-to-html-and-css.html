HTML Tags

>Now let's sink our teeth into HTML. This language provides the basic framework of a website. HTML stands for HyperText Markup Language.

When you add content to a page — such as a text, image, or video — it has to be placed inside tags. These tags tell the browser how to display content on a webpage.

A typical HTML element is comprised of the content and the enclosing tags. Each HTML tag has its own name, which is surrounded by angle brackets. For example, the tag for main headings is called h1 (heading 1, i.e. “first level heading”).

Tags in a code are like road signs. When you're entering a city, for example, you'll see a sign that reads "Welcome to San Francisco", and as you're leaving the city limits you'll drive past a sign that says "You are now leaving San Francisco". Much like road signs that have a city's name on it, both the opening and closing tags will contain the name of that tag. The resulting element is typically referred to by the name of its tag.

Headings

There are six different sizes for headings in HTML: h1, h2, h3, h4, h5, and h6. The concept of heading sizes comes from books, which have the main heading on the title page (h1), chapter headings (h2), section titles (h3), and smaller segments (h4, h5, h6).

As the biggest of the bunch, h1 is used for the most significant title, while h6 represents the least important and is even smaller in stature than standard text. We sort headings according to their relevance, and this pecking order makes it easier for search engines to decipher websites. Moreover, readers usually find it easier to understand webpages arranged in such a manner.

n general, h1 is used all the time. You can think of it as the headliner at a music festival, with h2 and h3 the supporting acts. Then you have h4 and h5, who play on the smaller stages, while h6 is that band you've never heard of.

Attributes

Attributes can provide additional information about an element, with different attributes being used for different types of information.

An attribute’s name tells the browser which property to apply, while its value contains the specific details of said property. For example, the value of the href attribute is the address of the resource that it should take us to:

An attribute must be put inside the opening tag. The value of an attribute can be put either in single href='http://info.cern.ch' or double href="http://info.cern.ch" quotation marks. Either is fine, just make sure that you use the same type of quotation mark consistently to make your code more readable.

For the < a > tag, href is a required attribute, and a link will not function without it. We can also add optional attributes if we want to, such as the target attribute, which tells the browser how to open a link. For instance, it can be opened in the same tab or a brand new one.

Since a link will work without it, using the target attribute is optional. However, if a link takes us to another website, it’s better to open it in a new tab. This way, your page will stay open in the user’s browser, and you won’t lose traffic.

When there are multiple attributes in the opening tag of your element, you should separate them using the space bar. The order in which they appear doesn’t matter.
